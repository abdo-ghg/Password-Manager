Password Manager Deployment Requirements

1. Runtime Requirements
- OS: Windows 10/11, Linux, or macOS
- Python: 3.10 or newer
- Package manager: pip (latest recommended)
- App server: Streamlit
- Browser: Any modern browser (Chrome, Edge, Firefox)

2. Python Dependencies
- streamlit
- bcrypt

3. Project Structure Requirement
- The app must be started from the App folder:
  - App/app.py
  - App/authentication.py
  - App/file_management.py
  - App/password_checker.py
  - App/password_generator.py
- Data files are created automatically in the current working directory:
  - users.csv
  - passwords.csv

4. Local Deployment Steps
1. Open terminal in project root.
2. Create and activate virtual environment:
   - Windows:
     - py -m venv .venv
     - .\.venv\Scripts\activate
   - Linux/macOS:
     - python3 -m venv .venv
     - source .venv/bin/activate
3. Install dependencies:
   - pip install -r requirements.txt
4. Run app:
   - cd App
   - streamlit run app.py

5. Production Deployment Notes
- Set working directory to App before launching Streamlit.
- Start command:
  - streamlit run app.py --server.port $PORT --server.address 0.0.0.0
- Ensure write permissions for app directory (CSV files are created/updated at runtime).
- For persistent storage, mount a persistent volume or move storage to a database.

6. Security Notes
- Current password storage uses base64 encoding in file_management.py; this is not strong encryption.
- For production, replace CSV storage and base64 with a secure database and real encryption.
- Keep the deployment private unless TLS/HTTPS and stronger secret management are configured.
